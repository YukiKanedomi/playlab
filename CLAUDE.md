# Playlab（遊びの実験室）

思いつきのゲームをどんどん形にして、**1リポジトリ（モノレポ）**で公開・試遊できる実験場。
詳しい構想は `NOTES.md`（=Yorozuyaで詰めた引き継ぎメモ）を参照。本ファイルは作業時に必ず効く原則。

## 最上位の原則：楽しく遊ぶ・遊び心
- 何よりまず**「遊んで楽しいか」**。技術・構成は手段。迷ったら「で、これ遊んで楽しい？」に立ち返る。
- 遊び心を全体に（作品本体／ハブ／演出／小さなインタラクション／隠し要素）。
- 完璧さより**まず動かして遊ぶ**（思いつき→即試遊）。手触り(juice)・気持ちよさは「楽しい」の核。

## 自由を縛らない
- **アプローチは何でもOK**（オリジナル／既存の模写学習／実験／改変・組合せ／ネタ一発…一例に過ぎず限定しない）。発想の自由が最優先。
- 各ゲームは**完全独立**：独自のフォルダ・依存・構造・作風で自由に。`shared/` は「使ってもいい道具箱(opt-in)」で、従うべき枠ではない（使わず作ってもOK）。
- **共通化のために自由を削らない**。共通化・スキル化は「同じことを2〜3回やったら切り出す」後追いで。

## ノウハウ再利用（任意）
- `shared/`＝Playlab内のコード再利用（薄く・任意・後付けで育てる）。
- **汎用に役立つものは積極的にスキル化**して `~/.claude/skills/` へ（全PJ横断で効く＝他PJへの技術波及の本命）。`skill-creator` で作成。
  - 候補: game-scaffold / game-juice / web-game-deploy / game-clone-study / asset準備(web-free-images) / game-playtest(render-check)。

## デザイン（没個性を避ける）
- **“AIっぽい没個性”を避ける**：絵文字の多用、ありがちなグラデ/カード乱用/紫系、量産フォント、無意味な装飾はしない。意図と個性のある洗練デザインを狙う。
- **世の中の良い実例をふんだんに参考**（受賞インディーのUI/サイト、名作のタイトル・HUD、洒落たLP）。「なぜ良いか」を言語化して取り込む。
- ツール: `frontend-design` / `theme-factory` / `render-check`(複数案を目視比較) / `web-free-images`(実素材)。

## 誠実さ（嘘をつかない）
- 既存作を参考にする場合は**学習目的**に留め、**商標名・公式アート/音源/ロゴはコピーしない**。仕組みを参考にしつつ絵・音・名前は自作/フリー素材。「〜に学ぶ習作」とクレジット。公開時は特に注意。

## 技術 / アーキテクチャ
- **Vite + 作品ごと自由**（共通utilsは持ちつつ各ゲームは Phaser/three.js 等を任意使用）。
- **公開は GitHub Pages**：`base: '/playlab/'`、データ取得は `import.meta.env.BASE_URL` 起点、**マルチページビルドで1pushで全公開**。
- スマホ前提：**タッチ操作・レスポンシブ**を標準に（`shared/input` のタッチ対応を最優先）。
- 構成（叩き台）:
  ```
  index.html (ハブ) / games.json (作品レジストリ・データ駆動)
  shared/ (loop・input・audio・save・juice・math ※任意)
  games/<slug>/ (各作品。index.html + main、任意で独自依存)
  vite.config.ts (base:'/playlab/'、multi-page input)
  ```

## 着手の一歩
- 小さく：**ハブ + games.json + shared薄基盤 + サンプル1本（スネーク等）** → GitHub Pages を通して「push→公開→スマホで試遊」を確立。
- 雛形フォルダ（new-game-template）を用意し、以後はコピーで量産。
